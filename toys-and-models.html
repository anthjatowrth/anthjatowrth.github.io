Toys and Models

Exploration d’une base relationnelle, création de tables/vues orientées BI, puis construction d’un dashboard Power BI interactif.

SQL Modélisation BI Power BI DAX Reporting
Repo GitHub Dashboard / PDF

Contexte

Toys and Models est un projet de reporting décisionnel basé sur une base de données transactionnelle (clients, commandes, produits, paiements). L’objectif : fournir une lecture claire de la performance commerciale via des KPIs fiables et filtrables.

(Tu peux remplacer ce texte par le contexte exact de ton projet.)

Objectifs

  • Comprendre le modèle relationnel et fiabiliser les jointures clés.
  • Créer des vues SQL “reporting-ready” (granularité, agrégations, dimensions).
  • Concevoir un modèle BI (type étoile) et des mesures DAX.
  • Produire un dashboard interactif (filtres, drill-down, KPIs).

Travail SQL

Après exploration du schéma, j’ai construit un ensemble de vues SQL dédiées au reporting : CA par période, performance par catégorie/produit, segmentation client, suivi des commandes et paiements.

  • Jointures multi-tables (orders, orderdetails, customers, products, payments…)
  • Agrégations (mensuelles / trimestrielles), cohérence des totaux
  • Calculs de marges, panier moyen, top produits / top clients
  • Structuration d’un modèle de type étoile pour la BI

Power BI

Les vues ont été importées dans Power BI afin de construire un modèle analytique, puis des mesures DAX pour rendre les KPIs comparables dans le temps.

  • Modèle relationnel + tables de dates
  • Mesures DAX : CA, marge, croissance, panier moyen
  • Filtres, segments, drill-down par produit/client/pays
  • Pages dédiées : vue exécutive, ventes, produits, clients

Résultats & apprentissages

Fiabilité

Vues SQL reproductibles, totaux cohérents, logique de calcul documentée.

Lisibilité

Dashboard orienté décision : KPIs, tendances, segments, comparaisons.

Scalabilité

Modèle pensé pour ajouter de nouveaux indicateurs ou dimensions.
